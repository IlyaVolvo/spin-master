üîê Authentication Endpoints

POST /auth/member/login Public
Member Login - Authenticates a member using email and password. Creates a session and returns member data with optional JWT token.

Request Body:

email (string, email) required
password (string) required

Response (200):

{ "token": "jwt_token_string", "member": { "id": 1, "email": "user@example.com", "firstName": "John", "lastName": "Doe", "roles": ["PLAYER"], "mustResetPassword": false, ... }, "sessionId": "session_id" }
POST /auth/member/logout Authenticated
Member Logout - Destroys the current session and logs out the member.

Response (200):

{ "message": "Logged out successfully" }
GET /auth/member/me Authenticated
Get Current Member - Returns the currently authenticated member's information from the session.

Response (200):

{ "member": { "id": 1, "email": "user@example.com", "firstName": "John", "lastName": "Doe", "roles": ["PLAYER"], ... } }
POST /auth/member/change-password Authenticated
Change Password - Allows a member to change their own password. Current password is optional if password was reset by admin.

Request Body:

currentPassword (string) optional - Required if password exists
newPassword (string, min 6 chars) required

Response (200):

{ "message": "Password changed successfully" }
POST /auth/member/:id/reset-password Admin Only
Admin Reset Password - Allows an admin to reset a member's password. If newPassword is not provided, sets password to empty to force setup on next login.

URL Parameters:

id (number) required - Member ID

Request Body:

newPassword (string, min 6 chars) optional - If omitted, forces password setup on next login

Response (200):

{ "message": "Password reset successfully" }

üíö Health Check

GET /api/health Public
Health Check - Simple endpoint to check if the API server is running.

Response (200):

{ "status": "ok" }

Notes:

  • All endpoints except /auth/member/login and /api/health require authentication.
  • Authentication is done via session cookies (preferred) or JWT token in the Authorization header.
  • Admin-only endpoints require the user to have the ADMIN role.
  • Organizer-only endpoints require the user to have the ORGANIZER role.
  • Members can edit their own email, phone, address, and picture. Admins can edit all fields.
  • Members cannot be deleted if they have played in any matches (excluding BYE matches).
  • Tournament participants are automatically deleted when a member is deleted.
  • Rating history is automatically deleted (cascade) when a member is deleted.
