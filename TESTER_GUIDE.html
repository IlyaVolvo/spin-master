Spin Master — Tester Guide

Overview

The Spin Master is a web application for managing a table tennis club. It handles member registration, tournament creation and execution across multiple formats, match result recording, and player rating tracking over time.

The system is built around a plugin architecture for tournament types — each format (Round Robin, Playoff, Swiss, etc.) is a self-contained plugin that drives its own creation wizard, match management, and completion logic.

Key capabilities:

  • Member management — add, edit, import/export players with roles, ratings, and contact info
  • Tournament creation — six supported formats, from simple round robins to multi-phase compound tournaments
  • Live tournament management — enter match scores, track standings, advance brackets
  • Ratings — USATT-style numeric ratings that update after matches and tournaments
  • Statistics & History — rating trends, head-to-head records, match chronology

2. Adding a Player

From the Players page:

  1. Click the Add Player button.
  2. Fill in the required fields:
    • First Name and Last Name — 2–50 characters, letters only (hyphens, apostrophes, spaces allowed between parts)
    • Email — must be unique across all members
    • Birth Date — within the allowed range (age 5–105)
    • Gender — Male, Female, or Other
    • Role(s) — at least one: Player, Coach, Admin, Organizer
  3. Optionally fill in:
    • Rating — integer 0–9999 (USATT-style)
    • Phone — valid US phone number format
    • Address
    • Picture URL
  4. Confirm and save.

The system creates the account and sends a password-setup invitation email to the new member. They complete onboarding via a one-time secure link.

3. Importing Players (Bulk)

This is the most efficient way to populate the club roster. The import accepts a CSV file.

CSV Format

Required columns:

ColumnNotes
firstname
lastname
emailMust be unique, valid format
birthdate or date of birthYYYY-MM-DD recommended; any parseable date format works

Optional columns:

ColumnNotes
genderMALE, FEMALE, or OTHER
rolesComma-separated first letters: P = Player, C = Coach, A = Admin, O = Organizer
phoneUS phone number format
addressFree text
ratingInteger 0–9999

Example CSV:

firstname,lastname,email,birthdate,gender,roles,phone,rating
John,Smith,john.smith@example.com,1985-03-15,MALE,"P, C",(555) 123-4567,1450
Jane,Doe,jane.doe@example.com,1990-07-22,FEMALE,P,,1200
Bob,Johnson,bob.j@example.com,1978-11-03,MALE,P,555-987-6543,1800

Lines starting with # are treated as comments and ignored.

How to Import

  1. Go to the Players page.
  2. Click the Import button.
  3. Select your .csv file.
  4. The system validates every row and shows a detailed results summary:
    • Successfully imported — count and list of created members
    • Failed — rows that did not pass validation, with per-row error details
    • Email issues — members created but whose invitation email could not be delivered
  5. You can toggle whether invitation emails are sent during import.

Preparing Data from Other Systems

If you have player data in another system (USATT database export, Omnipong, club spreadsheet, league management software, etc.):

  1. Export from the source system — most systems support CSV or Excel export.
  2. Open in a spreadsheet application — Google Sheets, Excel, LibreOffice Calc, or Numbers all work.
  3. Rename/map columns to match the expected header names: firstname, lastname, email, birthdate, and optionally gender, roles, phone, address, rating.
  4. Clean the data:
    • Ensure every row has a unique, valid email address.
    • Format birth dates as YYYY-MM-DD for best results.
    • Ratings should be plain integers (no commas, no text).
    • Phone numbers should be US format (10 digits, optionally with country code 1).
  5. Save/export as CSV (.csv file with UTF-8 encoding).
    • Google Sheets: File → Download → Comma Separated Values (.csv)
    • Excel: File → Save As → CSV UTF-8
    • LibreOffice Calc: File → Save As → Text CSV, ensure UTF-8 charset

The system also supports exporting your current player list to CSV from the Players page, which you can use as a template for the import format.

Duplicate Handling

The import checks for duplicates by email and by first name + last name combination. Duplicate rows are rejected with a clear error message — existing members are never overwritten.

4. Tournament Types

All tournaments are created from the Players page:

  1. Click to start tournament creation.
  2. Select a tournament type from the menu.
  3. Optionally name the tournament.
  4. Select participating players (only active members are available; use filters, Select All / Deselect All as needed).
  5. Continue — the rest of the flow is type-specific and guided by the creation wizard for that format.

Round Robin

Every player plays every other player once. Best for smaller groups (3+ players).

  • Full all-play-all schedule is generated automatically.
  • Standings update as results are entered.

Playoff / Bracket

Single-elimination bracket. Requires 6+ players.

  • Seeded bracket structure with a preview before creation.
  • You can review and adjust seeding/bracket positions before confirming.
  • Winners advance through rounds until a champion is determined.

Multi Round Robin

Players are split into multiple separate round-robin groups (6+ players).

  • You configure group sizes and review group assignments (snake-draft seeding with drag-and-drop adjustment).
  • Each group runs as its own independent round robin.

Preliminary + Final Round Robin

Two-phase compound tournament (8+ players):

  • Phase 1: Players compete in preliminary round-robin groups.
  • Phase 2: Top qualifiers from each group advance to a final round-robin.
  • Supports auto-qualified players — highest-rated players skip preliminaries and go directly to the final.
  • Qualification order: auto-qualified → all 1st-place finishers → fill remaining spots from 2nd/3rd places by rating.

Preliminary + Final Playoff

Two-phase compound tournament (8+ players):

  • Phase 1: Preliminary round-robin groups (same as above).
  • Phase 2: Top qualifiers advance to a final playoff bracket.
  • Same auto-qualification and filling logic as Preliminary + Final Round Robin.

Swiss System

Multiple rounds of pairings based on performance (6+ players).

  • You configure the number of rounds.
  • Players are paired each round against opponents with similar records.
  • No one is eliminated — all players compete in every round.
  • Best for larger groups where a full round robin would be impractical.

5. Future Work

The following features are planned for future development:

Club Check-In via QR Scanner

  • Members will be able to check in to the club by scanning a personal QR code.
  • The system already supports QR token infrastructure on member accounts.
  • A scanner station (tablet or phone at the entrance) will read QR codes and register attendance.
  • This enables session tracking, attendance history, and automated club-night rosters.

Payment Integration

  • Pluggable payment methods connected to the system.
  • Support for membership dues, lesson payments, and event entry fees.
  • Integration with payment providers (Stripe, Square, etc.) for online and in-person transactions.
  • Payment status tracked per member with history and receipt generation.

Coach Schedules

  • Coaches will be able to publish their available time slots.
  • Schedule management integrated into the coach's profile.
  • Visibility for all club members to see coach availability.

Individual Lesson Scheduling

  • Members can browse available coach time slots and book individual lessons directly through the system.
  • Booking confirmation, reminders, and cancellation flow.
  • Lesson history tracked per member and per coach.

Group Lessons

  • Coaches can create group lesson sessions with capacity limits.
  • Members can sign up for open group lesson slots.
  • Waitlist support when sessions reach capacity.
  • Recurring lesson schedules for regular group sessions.
